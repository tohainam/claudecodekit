Overview

Claude Code Kit

AI-Powered Development Toolkit with intelligent agents, extensible skills, and automated workflows for modern software development.

3 Core Agents

Researcher, Scouter, Reviewer - specialized AI agents for different tasks

8 Extensible Skills

Domain knowledge with progressive disclosure design

7 Workflows

Scale-adaptive routing with human confirmation points

Key Principle: Brainstorm Before Automate - Interactive specification refinement through conversation before workflow execution.

Directory Structure

Project Structure 
.claude/
├── .specs/          # Finalized specifications
├── .reports/        # Research & analysis reports
├── .plans/          # Implementation plans
├── .state/          # Workflow execution state
├── agents/          # Agent definitions
├── skills/          # Skill modules
├── commands/        # Slash commands
└── rules/           # Global guidelines

Quick Start

Get up and running with Claude Code Kit in minutes.

1

Clone the Repository

bash 
git clone https://github.com/user/claude-code-kit
cd claude-code-kit
2

Start a Brainstorm Session

Begin by creating a specification for your feature through interactive conversation:

Claude Code 
/brainstorm user-authentication

This creates a spec file at .claude/.specs/user-authentication.md and starts an interactive session to refine requirements.

3

Execute the Workflow

Once your spec is finalized, run the full development workflow:

Claude Code 
/run feature user-authentication
4

Review Generated Artifacts

Check the output in designated folders:

.specs/ Finalized feature specifications
.reports/ Research and analysis reports
.plans/ Hierarchical implementation plans

Run Command

Execute development workflows with scale-adaptive routing and confirmation points.

Usage

Claude Code 
/run <workflow> <name>

Available Workflows

Workflow Requires Spec Phases Description
feature Yes research → plan → implement → test → review → commit Complete new feature development
bugfix No scout → debug → plan → fix → test → commit Bug investigation and fixing
hotfix No debug → fix → test → commit Urgent production fix
refactor No scout → plan → refactor → test → review → commit Safe code refactoring
research No research → scout → summarize Research without code changes
review No scout → review → report Independent code review
docs No scout → plan → generate → commit Documentation generation

If you don't specify a workflow, Claude will auto-detect based on your intent:

Skill

Debug

Systematic debugging and root cause analysis with 5-phase methodology.

Iron Law: NO FIXES WITHOUT ROOT CAUSE INVESTIGATION FIRST. Systematic debugging (15-30 min, 95% success) beats random fixes (2-3 hours, 40% success).

5-Phase Process

1

Investigate

Reproduce, gather evidence, check recent changes

2

Analyze

Find patterns, compare working vs broken code

3

Hypothesize

Form single hypothesis, test with minimal change

4

Fix

Write failing test first, implement minimal fix

5

Document

Record root cause, add regression test

Key Techniques

Technique Use When How
Git Bisect Regression bugs git bisect start && git bisect bad HEAD && git bisect good v1.0
5 Whys Root cause discovery Ask "Why?" 5 times until systemic issue found
Binary Search Large codebases Comment out half, test, repeat on failing half
Delta Debugging Minimize reproducer Remove code until minimal failing case

Stop Signals

These thoughts indicate deviation from systematic process:

  • "Quick fix for now, investigate later"
  • "Just try changing X and see"
  • "Add multiple changes, run tests"
  • "3+ failed fix attempts" → Escalate to architecture review
