Core Concepts

Understanding the architecture of Claude Code Universal Config

‚ö°

Commands

+

Commands are slash commands you use to trigger specific workflows. They're your main interface for interacting with Claude Code.

How Commands Work

Type a command like /feature or /bugfix to start a structured workflow. Each command orchestrates multiple agents and follows best practices defined in rules.

Command Categories

  • Workflow Commands: /feature, /bugfix, /plan, /implement
  • Quality Commands: /review, /test, /refactor, /debug
  • Git Commands: /commit, /pr
  • Setup Commands: /onboard, /discuss, /check, /mcp-register

See all 14 commands ‚Üí

üéØ

Skills

+

Skills are domain-specific knowledge modules that are auto-loaded based on semantic matching with your task context.

Auto-Loading Behavior

When you work on a task, Claude analyzes the context and automatically loads relevant skills. For example, working on tests? The testing skill loads automatically.

Available Skills

  • architecture - System design and patterns
  • code-quality - Best practices and clean code
  • debugging - Root cause analysis
  • documentation - Writing effective docs
  • git-workflow - Commits, branches, PRs
  • performance - Optimization strategies
  • project-analysis - Codebase analysis for /onboard
  • refactoring - Safe code improvement
  • security-review - Security auditing
  • skill-creator - Creating new skills
  • testing - TDD and test strategies
ü§ñ

Agents

+

Agents are specialized sub-agents with specific roles in the workflow. Commands orchestrate agents to complete complex tasks.

How Agents Work

When you run a command like /feature, it uses multiple agents: planner creates the plan, implementer executes it, test-writer adds tests, and code-reviewer checks quality.

Agent Roles

Agent Model Purpose
facilitator opus Leads discussions, creates ADRs
planner opus Creates detailed implementation plans
implementer sonnet Executes plans step-by-step
code-reviewer sonnet Reviews code quality and security
test-writer sonnet Writes comprehensive tests
debugger sonnet Finds root causes of issues
refactorer opus Improves code structure safely
doc-writer sonnet Writes clear documentation
security-auditor opus Audits for security vulnerabilities
üìã

Rules

+

Rules define coding standards, communication patterns, safety protocols, and workflow procedures. They're always active and guide Claude's behavior.

Rule Categories

  • code-quality.md - Naming conventions, structure, patterns, anti-patterns
  • communication.md - Response format, when to ask questions
  • safety.md - Protected files, destructive operations, confirmations required
  • workflow.md - Plan ‚Üí Code ‚Üí Test ‚Üí Review process
  • skill-loader.md - Auto-load skills based on task context
  • mcp-loader.md - Auto-use MCP servers when relevant
  • knowledge-freshness.md - Source priority chain, freshness attribution

Customization

You can add project-specific rules by creating new files in .claude/rules/. Rules with path filters apply only to matching files.

Command Reference

All 14 commands with usage examples and workflows

/onboard

Setup

Analyze project and recommend Claude Code setup

+

When to Use

Setting up Claude Code for a new project or updating configuration as your project evolves.

What It Does

  • Detects languages, frameworks, databases, testing tools, CI/CD
  • Generates ONBOARD-REPORT.md with full analysis
  • Recommends .claude/rules/ files to create
  • With --apply flag, creates recommended files automatically

Usage

/onboard
# Or with auto-apply:
/onboard --apply

Output

  • ONBOARD-REPORT.md with detected technologies
  • Recommended rules files (created with --apply)

/discuss

Workflow

Structured discussion for requirements and decisions

+

When to Use

Before /feature or /plan when requirements are unclear, architectural decisions needed, or trade-offs must be analyzed.

Workflow

Facilitator agent leads: Clarify ‚Üí Explore ‚Üí Decide ‚Üí Create Discussion Summary + ADR

Usage

/discuss "Should we use REST or GraphQL for the API?"
/discuss "How to structure authentication?"

Output

  • .claude/.discussions/[datetime]-[topic].md - Discussion summary
  • .claude/.decisions/[datetime]-[topic].md - ADR if decision made

/plan

Workflow

Create implementation plan only

+

When to Use

When you want to review the plan before implementation, or plan a complex task for later execution.

What It Does

Planner agent creates a detailed plan with phases, steps, file:line hints, and test strategy. Plan is saved to .claude/.plans/ for later use with /implement.

Usage

/plan "Refactor authentication module"
/plan "Add real-time notifications"

Output

  • .claude/.plans/[datetime]-[type]-[name].md
  • Plan includes: phases, steps, affected files, dependencies, risks

/implement

Workflow

Execute an existing plan

+

When to Use

After creating a plan with /plan, or to resume a partially completed plan.

What It Does

Implementer agent reads the plan, executes steps one by one, validates changes, updates checkboxes, and reports progress.

Usage

/implement .claude/.plans/2025-12-28-feature-auth.md
# Or use latest plan:
/implement

Process

  • Reads plan file and understands all steps
  • Executes each step with validation (lint, typecheck, tests)
  • Updates plan checkboxes as steps complete
  • Reports blockers if encountered

/review

Quality

Review code changes for quality and security

+

When to Use

Before committing, after implementation, or reviewing PRs.

What It Checks

  • Code quality (naming, structure, patterns)
  • Security (OWASP, input validation, auth)
  • Performance (obvious inefficiencies)
  • Tests (coverage, quality)
  • Documentation (comments, READMEs)

Usage

/review
/review src/auth/
/review --security-focus

Output

Review report with findings categorized by severity: BLOCKER, WARNING, SUGGESTION, PRAISE

/test

Quality

Write tests for code

+

When to Use

After implementing new features or fixing bugs, to add comprehensive test coverage.

What It Does

Test-writer agent creates unit, integration, and E2E tests following the testing pyramid and TDD principles.

Usage

/test src/services/user-service.ts
/test --tdd "authentication feature"

Test Types

  • Unit tests - Business logic, edge cases
  • Integration tests - API endpoints, database
  • E2E tests - Critical user flows (optional)

/debug

Quality

Debug issue and find root cause

+

When to Use

When you need to understand why something is broken, without fixing it yet. For diagnosis only (use /bugfix to fix).

What It Does

Debugger agent analyzes code, traces execution flow, identifies root cause, and provides detailed diagnosis report.

Usage

/debug "Why is the API timing out?"
/debug "Memory leak in dashboard component"

Output

  • Root cause analysis
  • Affected code locations
  • Reproduction steps
  • Recommended fixes (but doesn't implement them)

/refactor

Quality

Refactor code safely

+

When to Use

Improving code structure, removing duplication, applying design patterns, or cleaning up tech debt.

What It Does

Refactorer agent improves code while maintaining behavior. Tests run after each change to ensure no regressions.

Usage

/refactor src/utils/helpers.ts
/refactor "Extract API client logic"

Safety Measures

  • Tests must exist and pass before refactoring
  • Changes made in small, atomic steps
  • Tests run after each step
  • Rollback if tests fail

Getting Started

+

Set up Claude Code Universal Config and run your first command in under 5 minutes.

Step 1: Install the Configuration

Clone the repository or copy the .claude/ directory to your project:

# Clone the repo
git clone https://github.com/your-org/claude-code-universal-config.git

# Copy .claude directory to your project
cp -r claude-code-universal-config/.claude /path/to/your/project/

Step 2: Analyze Your Project

Run the onboarding command to detect your tech stack and get recommendations:

/onboard --apply

This will create an ONBOARD-REPORT.md and generate recommended rules files for your stack.

Step 3: Run Your First Command

Try implementing a simple feature to see the workflow in action:

/feature "Add a hello world endpoint"

The planner will create a plan, ask for approval, then the implementer will execute it.

Step 4: Understand the Workflow

During the feature implementation, you'll see:

  • A detailed plan created in .claude/.plans/
  • Step-by-step implementation with validation
  • Options to add tests (say yes to learn testing workflow)
  • Code review findings
  • Option to commit changes (optional)

Next Steps

  • Read the Concepts section to understand Commands, Skills, Agents, Rules
  • Explore the Command Reference to see all available commands
  • Try Tutorial 2: Customization to adapt the config to your needs
2

Customization

+

Adapt Claude Code Universal Config to your project's specific needs.

Step 1: Add Project-Specific Rules

Create a new rule file for your project conventions:

# Create a new rule file
touch .claude/rules/my-project.md

Example content:

---
paths: src/**/*.ts
---

# My Project Rules

## API Conventions
- All API routes must have rate limiting
- Use Zod for request validation
- Return errors in { error: string } format

## Component Patterns
- Use React Query for data fetching
- Prefer server components in Next.js
- Use Tailwind CSS for styling

Step 2: Configure Permissions

Control what operations require confirmation by editing .claude/settings.json:

{
  "permissions": {
    "deny": ["Bash(rm:*)"],
    "allow": ["Bash(npm:install)"]
  },
  "hooks": {
    "enabled": true
  }
}

Step 3: Disable Optional Features

If you don't want certain workflows, you can remove or modify commands:

  • Remove command files from .claude/commands/ to disable them
  • Edit workflow rules in .claude/rules/workflow.md
  • Customize agent prompts in .claude/agents/

Step 4: Add Custom Skills

See Tutorial 3 for creating custom skills specific to your domain.

Git Ignore Local Settings

Create .claude/settings.local.json for personal overrides (already gitignored):

{
  "permissions": {
    "allow": ["Bash(my-custom-script:*)"]
  }
}

Create Your Own Skill

+

Add domain-specific knowledge that auto-loads when relevant.

Step 1: Use the Skill Creator

The easiest way is to let Claude create the skill for you:

# Claude will guide you through the process
"Create a new skill for handling GraphQL schemas"

The skill-creator skill will automatically load and help you structure the new skill.

Step 2: Skill Structure

A skill has this structure:

.claude/skills/my-skill/
‚îú‚îÄ‚îÄ SKILL.md              # Main skill file (always loaded)
‚îî‚îÄ‚îÄ references/           # Detailed docs (loaded on-demand)
    ‚îú‚îÄ‚îÄ quick-ref.md
    ‚îú‚îÄ‚îÄ patterns.md
    ‚îî‚îÄ‚îÄ tools.md

Step 3: Write SKILL.md

The main skill file should have:

# My Skill Name

## Quick Start
Brief overview and most common use cases

## Quick Reference
Tables, checklists, key patterns

## Reference Files
| File | Content |
|------|---------|
| [references/patterns.md](references/patterns.md) | Design patterns |
| [references/tools.md](references/tools.md) | Tool setup and usage |

Step 4: Update MANIFEST.md

Add your skill to .claude/skills/MANIFEST.md:

| my-skill | GraphQL schema design and best practices | .claude/skills/my-skill/SKILL.md |

(The skill-creator does this automatically)

Step 5: Test the Skill

Try a task that should trigger your skill:

"Design a GraphQL schema for user authentication"

Claude should auto-load your skill and you'll see: Skills Loaded: my-skill

Best Practices

  • Keep SKILL.md concise (< 500 lines) - use references/ for detail
  • Use tables and checklists for quick scanning
  • Write clear semantic descriptions for auto-loading
  • Include examples and code snippets
  • Test that the skill loads when expected

Installation

Get started in less than 1 minute

Method 1: Clone Repository

Clone and copy the .claude/ directory to your project:

git clone https://github.com/your-org/claude-code-universal-config.git
cp -r claude-code-universal-config/.claude /path/to/your/project/
cd /path/to/your/project
/onboard --apply

Method 2: Direct Download

Download and extract the .claude/ directory:

curl -L https://github.com/your-org/claude-code-universal-config/archive/main.zip -o claude-config.zip
unzip claude-config.zip
mv claude-code-universal-config-main/.claude /path/to/your/project/
cd /path/to/your/project
/onboard --apply

Verification

After installation, verify the setup:

# Check that .claude directory exists
ls -la .claude/

# Run onboarding
/onboard

# You should see ONBOARD-REPORT.md created
cat ONBOARD-REPORT.md

Requirements

  • Claude Code (desktop or CLI)
  • Git (for /commit and /pr commands)
  • GitHub CLI (gh) for /pr command (optional)
